Mapping and scaling actions experiments Design history

Understanding the design challenge

We began with research, analysis and ideation to explore how we could scale the previous work on mapping and actions (opens in new tab) if a user had 70+ actions to choose from (dependent on scheme offer, available area calcuation, land cover types and actions from previous agreemments), some of which would be incompatible with each other.

We also needed to improve the usability of the map component to ensure users could easily navigate to, and interact, with their land parcels. We collaborated with the Mapping Improvements team to learn from their recent user testing insights on this full-width layout and understand mutual user needs across our services.

The design approach

A key focus was alignment with the Mapping Improvements team, including exploration of a full-width, fluid layout to improve map usability and make better use of screen space. Several layout concepts were tested, including a 20/80 column structure across two rows to support multiple farms, contextual parcel information and action filtering alongside the map interface.

The work also explored how 2026 SFI action compatibility rules could be surfaced through inline messaging, helping users understand why certain actions could not be combined within the same parcel while testing scalability and usability across increasingly complex rule sets.

To support this, the team used Land Management System (LMS) “Land Cover” data extracts to display parcel information and derive compatible actions. “Land Cover Class” was later discounted following discussions with the Rural Payments Agency due to data quality concerns, with “Land Cover” identified as the more reliable and widely used dataset.

This work enabled exploration of edge cases such as multiple land covers within a single parcel and the resulting volume of compatible actions presented to users. These findings informed “How might we” ideation focused on helping farmers quickly identify relevant actions.

Additional work explored findability improvements through “Theme” filters and sort functionality, allowing users to organise actions by criteria such as payment rate and frequency of use, helping reduce cognitive load within complex land management journeys.

What we tested, outcomes and decisions

How might we improve map usability while ensuring users still easily find their desired parcels and actions from a long list?

Hypothesis 1:

If we provide a full width design for mapping

Then this negates the need for parcel links within the page due to increased map usability - eg, ability to see whole farm within map view, parcels not being obscured outside of map component when zooming, tool tips no longer hiding neighbouring parcels, easier navigation between parcels within map

We'll know this is true when:

  • The user is quickly able to choose specific parcels within a map
  • The user can navigate around the map from one parcel to another without excessive zooming in/out, closing of tooltips
  • The user does not seek alternative forms of navigation within farms/parcels due to task failure within scenario

Proven

Users were able to see the whole farm within the map view and parcels were not obscured outside of the map component when zooming. Users were able to navigate around the map and select parcels without excessive zooming in and out.

Tool tips no longer hid neighbouring parcels when they were opened due to the increased map size design.

Users did not seek alternative forms of navigation within farms/parcels due to task failure within the scenario (one user verbalised a preference for being able to see parcel list but was able to complete the task quickly without it).

Design conclusion

Aligning with the design and findings (password: 8PA38P) from the Mapping Improvements Team (now within Single Front Door team) has improved the usability of the map and actions page due to increased page real estate. This has negated the need for parcel links as default.

In line with accessiblity and progressive enhancement principles, we will provide parcel links as the navigation method for users who require this.

Hypothesis 2:

If we change jump tags to filters (for finding actions within long lists),

Then the user won't get lost within a long list of actions if they don't know the category their actions is within (see many parcels, many actions work)

We'll know this is true when:

  • 3 of 6 users did not demonstrate excessive scrolling when using the filters
  • 3 of 6 users can select and deselect filters to view subsets of the actions list (parallel to the filters) without excessive scrolling

Inconclusive

Users who engaged with the filters understood intuitively how they worked and knew which category the action they required belonged to, even though they weren't actions they would usually undertake on their land.

Design conclusion

3 of 6 participants used the filters effectively (while others used the search predominantly) and so we'd like to explore this further with more users.

Hypothesis 3:

If we use filters categories aligned to the 'Find' tool,

Then users will recognise the same content convention and find desired actions more quickly due to prior category familiarity.

We'll know this is true when:

  • Users demonstrably recognise the filter categories (Areas of interest)
  • User can find desired actions (when eligible within parcel) quickly when using the filters

Inconclusive

3 of 6 users knew which category the action they required belonged to, even though they weren't actions they would usually undertake on their land. This indicated a level of familiarity with the categories.

We'd like to further test this in order to gain more data from a larger sample size.
