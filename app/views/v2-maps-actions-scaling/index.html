Early research, analysis and ideation

See this Mural board for Research, Business Analysis and Design thinking.

This works explores:

  • Closer alignment to mapping improvements team design work inclusing a full width, fluid layout for increased map usability
  • Using a 20/80 column layout over 2 rows to allow for multiple farms, contextual parcel information and filtering of actions.
  • Using the 2026 SFI actions and compatibility rules to explore scalability and usability of inline messaging to help users understand why they can't combine certain actions within the same land parcel
  • Using 'Land Cover' data extract from LMS to show users information on maps and derive lists of compatible actions withing a parcel. 'Land cover class' was disregarded after a meeting with the RPA due to data quality issues. 'Land cover' is the more granular and reliable descriptor. It's also used widely in gov.uk guidance and within the Mapping Improvements project. This work allows us to explore edge cases like multiple land covers within a parcel, and how many compatible actions would be displayed from this data. These edge cases define helpful parameters for the number of actions a farmer is likely to see using our build model, driving 'How might we' ideation work to help farmers find the actions they need quickly.
  • Using 'Theme' filters for findability within a long list of actions for a parcel
  • Using a sort function to order a list of actions within a parcel by payment rate, frequenct of use (based on historic data) and other criteria
