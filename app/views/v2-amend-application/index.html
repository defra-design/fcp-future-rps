Amend an application Version 2: 23 March 2026

One of the initiatives (FDG-74) was to allow a user to re-access a previously submitted application in order to make amendments.

Usability testing was carried out on Version 1.

This version is for reference for the developers and has a slightly different flow this is due to the team set up and who creates/owns each screen.

When going through the flow, it was realised that the 'view application' screen as a read only version of the information submitted by the applicant through the journey, is not the screen that would then be used to amend any parcel/action combinations for the amend journey. From a tech perspective, the amend journey re-uses the flow (and therefore screens) in the apply journey; which don't necessarily fit for an amend journey. It means that either the original 'Review your actions' screen, or the ideated editable version of the 'view application' screen, would need to be used for both journeys. As the latter does not fit in with the initial apply journey, it was decided to use the 'review your actions' screen in the amend flow. As this screen doesn't contain the details about the user, it was decided that the 'Confirm your details' screen would need to also be included, as all information pertaining to the application should be shown so the user is aware of what they submitted.

This was discussed with Clare, and although not ideal, this flow is sufficient for an initial iteration.
