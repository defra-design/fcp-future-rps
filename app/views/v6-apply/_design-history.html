Future application journey - V6: Mapping (continued) Design history

The following elements were observed during research to need usability improvements, and have been addressed as follows.

Usability issues identified and resolved

Farm selection

Selecting a farm did not remove the non-selected farm's name and details from view, which caused confusion. This has been corrected so that only the selected farm's information were displayed.

Also, guidance copy has been added to help the user understand how to use the map and desired actions in conjunction with each other.

Parcel names and IDs

Parcel IDs were not previously included. Thus could casuse issues for farmers if multiple parcels shared the same name, which was evidenced by some participants in research. Parcel IDs were therefore added to the parcel text links alongside parcel names to address this.

Parcel list below the map

The "show more parcels" link (conditional reveal) displayed a numeric value (e.g. "5 more"), which users found confusing. This has been removed so the link simply indicates that additional parcels are available.

Visibility of actions from previous agreements

Users had difficulty locating actions from previous agreements, particularly when interacting with parcels on the map, resulting in excessive scrolling. This was addressed by improving the discoverability of existing actions in relation to map interactions, without compromising map usability.

Answers summary and previous agreement actions

Users were unclear about the distinction between parcels with actions from previous agreements and parcels updated in the current session. The visual hierarchy within the accordion did not sufficiently emphasise section headings. This was resolved by strengthening the hierarchy so that section headings were clearly distinguishable from their content.
