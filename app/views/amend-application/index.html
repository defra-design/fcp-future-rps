Amend an application (Feb 2026)

One of the initiatives (FDG-74) was to allow a user to re-access a previously submitted application in order to make amendments.

The scenario used for usability testing is that a user has previously submitted an application, consisting of four parcels with some actions. One of the parcels requires a SSSI consent. After submiting the application the user looks into obtaining a consent, and decided that it would take too long and so wants to remove that parcel from their application.

The application has not yet been picked up by RPA caseworkers.

The flow starts with the user logging in to the service and landing on the confirmation screen for submitting an application, as this is the latest point in the journey they reached.
